АНИМАЦИЯ
ПРЕИМУЩЕСТВА CSS-АНИМАЦИИ
Не нужны знания javascript
Cоздавая анимацию с помощью css экономятся рессурсы для ее воспроизведения, так как кода по сути там нету, а то что создает браузер воспроизводится движком
Контроль за последовательность кадров анимации так же ложиться на браузер из-за этого повышается производительность и эффективность веб-сайта.
КОНФИГУРИРОВАНИЕ АНИМАЦИИ
Для создания анимации в файле стилей в определенном селекторе необходимо создать свойство animation: В самом свойсве нужно будет указать несколько конфигурирующих параметров: имя анимации (обязательно), длительность анимации, ускорение, повтор и прочие. После этого нужно создать набор так называемых кадров с помощью правила @keyframes . В наборе кадров нужно указать то же имя что и для анимации будет выглядеть так - @keyframes name {список правил} . Список правил и будет обеспечивать анимацию.
animation-name
Определяет имя @keyframes, настраивающего кадры анимации
animation-duration
Определяет время, в течение которого должен пройти один цикл анимации
animation-timing-function
Настраивает ускорение анимации
animation-delay
Настраивает задержку между временем загрузки элемента и временем начала анимации
aanimation-play-state
Позволяет Вам приостановить и возобновить анимацию
animation-name
Определяет имя @keyframes, настраивающего кадры анимации
animation-fill-mode
Настраивает нужные значения, используемые анимацией, до и после исполнения
ОПРЕДЕЛЕНИЕ ПОСЛЕДОВАТЕЛЬНОСТИ  АНИМАЦИИ
После настройки основних свойств нужно непосредственно настроить @keyframes . В @keyframes есть 2 ключевых свойства from и to Они ознаменовывают собой начало и конец анимации - соответсвенно их числовые значения равны 0% и 100%, где 0% - это начало анимации а 100% - её конец. Таким образом можно настраивать процентное соотношение в анимации. Указав опредленные значения свойств, браузер сам изменит из от того что было указано в начале до того что было указано в конце за время анииации, которое указано в свойсве animation опереленного селектора.
СКОЛЬЖЕНИЕ ТЕКСТА
Это один из самых простых примеров, в нем показано скольжение текста в элементе от правого края блока. Из-за такого проигрывания анимации, контент может увиличиватся в размери и делать блок шире или уже и выше, либо меньше. В стиле параграфа указано что анимация будет длится 3 секунды, а имя расскадровки, на которое будет ссылкатся данная анимация - slidein. В селектор для параграфа можно укзаать и другие слити они не будут как-то влиять на проигрывание анимации, а только зададут цвет, позиционирование и прочие свойства. В @keyframes slidein укано всего два кадра from и to В первом кадре параграф будет с отсупом слева в 100% и шириной 300%, но за время длительности анимации, которое мы указали, значения изменяться и отсут пропадет вовсе, а ширина будет равна шырине родительского блока, а именно 100%. Эти значения указаны во втором кадре, и они и означают конец анимации. Показательный пример был отобран с рессурса: https://developer.mozilla.org/ru/docs/Web/CSS/CSS_Animations/Ispolzovanie_CSS_animatciy 
    p {
        animation duration: 3s;
        animation name: slidein;
    }


    @keyframes slidein {
        from {
            margin left: 100%;
            width: 300%;
        }


        to {
            margin left: 0%;
            width: 100%;
        }
    }

ВИДЫ СЕЛЕКТОРОВ
Селекторы это именованые значения с помощью каких можно придавать стилизации странице и ее отдельным частям. Так селектором может выступать класс элемента, его айди, можно создавать селекторы по набору тегов, так же селекторами могут выступать псевдокласы и всевдоэлементы, с помощью первых можно описывать стили при определенных событиях а с помощью последних создавать доп элементы.
УНИВЕРСАЛЬНЫЙ СЕЛЕКТОР
Универсальных селектор обозначен звездочкой - * и указывает что стили заключеныйе в скобки с данным селектором будет применяется ко всем (любым) элементам. Так же это селектор можно исопользовать и с псевдоэлементами например * :after {CSS-стили} , * :checked {CSS-стили}
СЕЛЕКТОР ЭЛЕМЕНТА
Данный вид селекторов обозначает приминение стилей по тегу Так например можно выбрать форматирование для всех изображений на сайте Выглядеть это будте следующим образом: img { width: 200px; height: 200px; } Таким образом мы указываем что все изображения, которые определены на странице через тег <img> будут квадратом с размером 200Х200 пикселей.
СЕЛЕКТОР КЛАССА
Самый распостраненный тип селектора С его помощью можно сократить количество места в HTML. Я говорю о том что одним из первых вариантов применения стилей к элементу было свойство styele , где строкой перечилсялся набор свойств, которые применялись к элементу. Селектор же класа позволил заключить набор стилей в файле с расширением .css под определенным наименованием, а это наименование указать в HTML елеенте с помощью атрибута class . Так например для создания класса в файле стилей создается селектор класса : class-name { ...styles }, затем файл стилей импортируеться в HTML документ, или же если клас создан в теге <style> то импортировать ничего не нужно. Затем в выбранном DOM елементе необхадимо указать атрибут class сдедующим образом <div class="class-name"> . Теперь применять это класс можно и для других элементов таким же способом, секономив пространство и время.
СЕЛЕКТОР ПОТОМКА
Селекторы потомков применяют стили к элементам, расположенным внутри элемента-контейнера. Например, ul li {text-transform: uppercase;} — выберет все элементы li , являющиеся потомками всех элементов ul. Например, если нужно отформатировать определенные потомки определенного элемента, этому элементу нужно задать стилевой класс: p.first a {color: green;} — данный стиль применится ко всем ссылкам, потомкам абзаца с классом first; p .first a {color: green;} — если добавить пробел, то будут стилизованы ссылки, расположенные внутри любого тега класса .first, который является потомком элемента <p> ; .first a {color: green;} — данный стиль применится к любой ссылке, расположенной внутри другого элемента, обозначенного классом .first
ДОЧЕРНИЙ СЕЛЕКТОР
Дочерний элемент является прямым наследником содержащего его элемента. Естественно, у одного элемента может быть несколько дочерних элементов,в то время как родительский элемент у каждого элемента может быть только один. Дочерний селектор позволяет применить стили только если дочерний элемент идёт сразу за родительским элементом и между ними нет других элементов, то есть дочерний элемент больше никуда не вкладывался. Например, p > strong выберет все элементы strong, которые являются дочерними по отношению к элементу p.

СЕСТРИНСКИЙ СЕЛЕКТОР
Сестринские отношения возникают между элементами, которые имеют в свою очередь общего родителя. Селекторы сестринских элементов могут помочь выбрать элементы из группы элементов одного уровня. Например, h1 + p — выберет все первые абзацы, идущие непосредственно за любым тегом <h1> , не затрагивая остальные абзацы; h1 ~ p — выберет все абзацы, которые являются сестринскими по отношению к любому заголовку h1 и идущие сразу после него.

CSS-ГРАДИЕНТ
CSS-градиенты это возможность сделать фон из двух и более цветов, которые плавно переходят из одного цвета в другой. CSS-градиенты уже достаточно давно применяются и поэтому имеют довольно неплохую поддержку браузерами. Большинство современных браузеров понимает их без префиксов, для IE9 и старше есть Gradient Filter, также для IE9 можно использовать SVG. Градиенты очень универсальны и могут быть использованы везде, где используются картинки: в фонах, в качестве буллетов списков, они задаются в content или border-image .
ЛИНЕЙНЫЙ ГРАДИЕНТ
Линейный градиент можно создать при помощи минимум двух цветов, у которых задано направление или линия градиента. Если направление не указано, то тогда используется значение по умолчанию — сверху-вниз. Цвета градиента по-умолчанию будут распределятся равномерно в направлении, перпендикулярном линии градиента. 

div {
    height: 200px;
    background: linear-gradient(45deg, #EECFBA, #C5DDE8);
}
С помощью ключевых слов таких как: to top, to right, to bottom, to left , которые в свою очередь соответствуют углу градиента, равному 0 deg , 90 deg , 180 deg и 270 deg соответственно. Для того, чтобы цвета неравномерно распределялись указывается начальная позиция каждого цвета через точки остановки градиента (color stops) . Color stops задаются в % , где 0% 0% — начальная точка, 100% — конечная точка, например: 

div {
    height: 200px;
    background: linear-gradient(to top, #E4AF9D 20%, #E4E4D8 50%,
    #A19887 80%);
}
РАДИАЛЬНЫЙ ГРАДИЕНТ
Главное отличие радиального градиента от линейного - цвета выходят из одной точки (центра градиента) и равномерно распределяются наружу, рисуя форму круга или эллипса. Форма градиента определяется ключевыми словами в соответствии геометрической форме circle или ellipse. Важно! Если форма не задана, по умолчанию радиальный градиент принимает форму эллипса. 


div {
height: 200px;
background: radial-gradient(white,
#FFA9A1);
}
Позицию центра можно задать с помощью ключевых слов, которые должны использоваться в свойстве background-position, с добавлением приставки at . Если позиция центра не задана, используется значение по умолчанию at center. С помощью значений, указанных в единицах длины: %, em или px, можно управлять размером эллипсообразного градиента. Первое значение задает ширину эллипса, второе — высоту. 

div {
height: 200px;
background: linear gradient(37deg,
#f25656, #18258c, #8c1864);
background size: 600% 600%;
animation: gradient 1s ease infinite;
}
Размер градиента задаётся с помощью ключевых слов. Значение по-умолчанию farthest-corner (к дальнему углу)
closest-side
Размер градиента рассчитывается из расстояния до любой ближней стороны блока для circle или до ближних сторон по X и по Y для ellipse
farthest-side
Размер рассчитывается из расстояния до дальних сторон
closest-corner
Размер рассчитывается из расстояния до ближних углов
farthest-corner
Размер рассчитывается из расстояния до дальних углов.
С помощью радиального градиента можно создавать реалистичные объёмные фигуры, такие как мячи, кнопки 
div {
width: 200px;
height: 200px;
border radius: 50%;
margin: 0 auto;
background:
radial-gradient(circle at
65% 15%, aqua, darkblue);
}
ПОВТОР ГРАДИЕНТА
В добавление к линейному и радиальному градиентам существует еще и повтор градиента, который можно задать всего лишь с небольшой помощью функций repeating-linear-gradient() и repeating-radial-gradient() соответственно. Увы, но фон из повторяющихся градиентов выглядит неаккуратно, поэтому желательно начинать следующий цвет с точки остановки предыдущего, создавая таким образом полосатые узоры.

background:
repeating-linear-gradient(45deg, #606dbc,

ФУНКЦИЯ CALC()
СALC () - это функция CSS, которая предлагает возможность расчитать значения свойств CSS. Данная функция используется везде, где есть даже минимальная возможностьприменить <length>,<frequency>, <frequency>,<angle>,<time>,<number>,или <integer> Эта функция принимает в качестве параметра математическое выражение, результат вычисления которого можно использовать как значение CSS Выражениеможетвключатьоператоры+,-,*,/нотолькоприусловии использованиястандартныхправилприоритетаоператоров. При помощи calc() можно задать позиционирование объекта с помощью отступа. В данном примере представлено как создается баннер занимающий в ширину всего окно с отступами по краям в 40px. 

.banner {
    margin-top: 20px;
    width: calc(100% - 80px);
    border: solid black 1px;
    box-shadow: 1px 2px;
    background-color: orange;
    padding: 6px;
    text-align: center;
    grid-column-start: 5;
    grid-column-end: 9;
}
3D ТРАНСФОРМАЦИЯ
CSS3 3D-трансформации позволяют создавать объемные, а главное реалистичные эффекты на веб-страницах. 3D-трансформации работают по подобию 2D-трансформаций разница состоит лишь в том, что элементы могут так же перемещаться вдоль оси Z, вглубь экрана и из него. Чтобы активизировать 3D-пространство, достаточно просто использовать свойство perspective для родительского контейнера.
Установка 3D-перспективы (perspective)
В обыкновенном потоке элементы отображаются плоскими и в той же плоскости, что и блок, содержащий их. Двумерные функции преобразования помогают изменять внешний вид элемента, однако этот элемент будет так же отображается в той же плоскости, что и содержащий его блок. Свойства perspective и perspective-origin можно использовать для добавления ощущения перспективы в сцене, делая элементы выше по оси Z (той есть ближе к смотрящему) и кажущимися большими, а те, которые находятся дальше — меньшими. Масштаб пропорционален d / (d - Z), где d — значение перспективы, является расстоянием от плоскости рисования до предполагаемого положения глаза зрителя. Если 3D-перспектива задается с помощью функции perspective(), 3D-пространство используется только для одного элемента. Свойство perspective помогает активизировать 3D-пространство внутри элемента, который в свою очередь содержит дочерние трансформированные элементы и применяется к ним. Данное свойство не наследуется.
z-index
Задает расстояние по оси Z. Значение может быть любым положительным числом. В случае если единица измерения не указана,то по-умолчанию единицой измерения считают вpx. Чем больше значение, тем менее выражен данный эффект.Значение в позиции 0 означает что перспектива отсутствует.
none
Значение указано по- умолчанию. Означает,что перспектива полностью отсутствует.
inherit
Значение наследует свои свойства лишь от родительского элемента
:nth-child(odd)
нечётные дочерние элементы
Чтобы задать точки трансформации для 3D-элемента perspective-origin нужно помнить, что изначально свойство устанавливает точку начала координат для свойства perspective. А вот значение perspective-origin: 50% 50%, по-умолчанию. Данное свойство позволяет изменять направление трансформации дочернего 3D-элемента. Свойство должно использоваться только вместе со свойством perspective для блока-контейнера и свойством transform для дочернего элемента. Данное свойство не наследуется.
значение по оси X/значение по оси Y
X - это первое значение; указывает положение элемента по оси. Задается в единицах длины, в % или одним из трех ключевых слов: left (эквивалентно 0% по оси X), center (эквивалентно 50% по оси X) или right (эквивалентно 100% по оси X). Y - это второе значение; указывает положение по оси. Задается в единицах длины, в % или одним из трех ключевых слов: top (эквивалентно 0% по оси Y), center (эквивалентно 50% по оси Y) или bottom (эквивалентно 100% по оси Y).

ШРИФТЫ
В CSS3 реализация сложных шрифтов обеспечивается посредством возможности @font-face , данную возможность можно применить с помощью следующих функций:
Требуемый шрифт (или, более вероятно, несколько версий шрифта для поддержки разных браузеров) загружается на сайт
Шрифт регистрируется в таблице стилей с помощью команды @font-face
Зарегистрированный шрифт используется в правилах стиля указанием его названия, точно так же, как и обычные веб-шрифты
Все современные браузеры поддерживают возможность @font-face , однако есть те, что не поддерживают определенные типы файлов шрифтов. Например, Internet Explorer , который в свою очередь поддерживает только файлы типа EOT (Embedded OpenType). Однако, данный формат EOT никогда ни пользовался большой популярностью и не используется никакими другими браузерами. Вместо него браузеры работают с — TTF (TrueType) и OTF (OpenType PostScript). Кроме этого, существуют еще два типа отображения шрифтов — SVG и WOFF. В таблице справа представлено краткое описание всех этих форматов шрифтов.
TTF (TrueType), OTF (OpenType PostScript)
Распространенные форматы шрифтов настольных компьютеров
EOT(Embedded OpenType)
Формат, специфичный для продуктов корпорации Microsoft. Не завоевал популярности у браузеров, за исключением Internet Explorer
SVG (Scalable Vector Graphics)
Универсальный графический формат, который можно использовать для создания шрифтов. Дает хорошие, но не отличные результаты — медленно отображается и демонстрирует текст пониженного качества
SVG (Scalable Vector Graphics)
Единый формат шрифтов будущего. Поддерживается новыми версиями браузеров
Семейство шрифтов: свойство font family
Cвойство font-family существует для выбора начертания шрифта. Из-за того, что невозможно предсказать, установлен тот или иной шрифт на компьютере у посетителя сайта, следует прописывать все возможные варианты однотипных шрифтов. В этом случае браузер будет проверять их наличие, последовательно перебирая предложенные варианты. Если в названии шрифта, например, будут иметься пробелы или символы (например, #, $, %), то следует заключаить их в кавычки. Это делается для того, чтобы браузер мог понять, где начинается и заканчивается название шрифта. Данное свойство не наследуется.
family-name
Название (имя) семейства шрифтов.Как пример можно рассмотреть: Times, Courier, Arial. Рекомендуется указывать вместе с базовым семейством.
generic-family
Базовое семейство. CSS определяет пять базовых семейств шрифтов: Шрифты с засечками — Serif (Times New Roman, Times, Garamond, Georgia) Рубленые шрифты — Sans-serif (Helvetica, Geneva, Arial, Verdana, Trebuchet, Univers) Моноширинные шрифты — Monospace (Courier, Courier New, Andele Mono) Рукописные шрифты — Cursive (Comic Sans, Gabriola, Monotype Corsiva, Author, Zapf Chancery) Аллегорические шрифты (Western, Woodblock, Klingon).
